As per OpenCVwe have first stable release in the 4.x line but yet not official release.Some key points 4.x line will bring to the table
    • 1. OpenCV is now C++11 library and requires C++11-compliant compiler.
    • 2. Minimum required CMake version has been raised to 3.5.1.
    • 3. New module G-API has been added, it acts as an engine for very efficient graph-based image procesing pipelines..
    • 4. dnn module was updated with Deep Learning Deployment Toolkit from the OpenVINO™ toolkit R4.
    • 5. dnn module now includes experimental Vulkan backend and supports networks in ONNX format.
    • 6. The popular Kinect Fusion algorithm has been implemented and optimized for CPU and GPU (OpenCL)
    • 7. QR code detector and decoder have been added to the objdetect module
Let's start, We will divide our installation process into 4 Steps:

  • 1. Core Library Installation
  • 2. Virtual Environment Setup
  • 3. Configuring Core Libraray With Virtual Environment
  • 4. Verification of Installation

Step 1 : Core Library Installation: Open terminal in root, run below commands
                # Please run below commands before we start installation process.

                $ sudo apt-get update
                $ sudo apt-get upgrade

                # Now start installing required packages.

                $ sudo apt-get install build-essential
                $ sudo apt-get install cmake git libgtk2.0-dev pkg-config libavcodec-dev libavformat-dev libswscale-dev
                $ sudo apt-get install python-dev python-numpy libtbb2 libtbb-dev libjpeg-dev libpng-dev libtiff-dev libjasper-dev
                  libdc1394-22-dev

                # Download OpenCV source code form git or
                # stable version form here https://opencv.org/releases.html and unpack it

                $ cd ~/myworking_directory # make sure to move in your working directory
                $ git clone https://github.com/opencv/opencv.git
                $ git clone https://github.com/opencv/opencv_contrib.git
Step 2 : Virtual Environment Setup: Working with virtual environments (venvs) in python helps us a lot , it helps us in maintaining different versions of the same package.

            $ sudo pip install virtualenv virtualenvwrapper
            $ sudo rm -rf ~/get-pip.py ~/.cache/pip
            $ mkvirtualenv cv -p python3 # here cv is our virtual environment name, we can give it any name.
Step 3 : Configuring Core Libraray With Virtual Environment: We have to update our ~./bashrc file

            $ echo -e "\n# virtualenv and virtualenvwrapper" >> ~/.bashrc
            $ echo "export WORKON_HOME=$HOME/.virtualenvs" >> ~/.bashrc
            $ echo "export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python3" >> ~/.bashrc
            $ echo "source /usr/local/bin/virtualenvwrapper.sh" >> ~/.bashrc
            $ source ~/.bashrc

            # Now open newly created virtual environment having name

            $ workon cv #  will open newnly created venv
            $ pip install numpy
            $ cd ~/opencv # move to directory where OpenCv is kept that we have downloaded form git
            $ mkdir build
            $ cd build
            $ cmake -D CMAKE_BUILD_TYPE=RELEASE \
              -D CMAKE_INSTALL_PREFIX=/usr/local \
              -D INSTALL_PYTHON_EXAMPLES=ON \
              -D OPENCV_ENABLE_NONFREE=ON \
              -D OPENCV_EXTRA_MODULES_PATH=~/opencv_contrib/modules \
              -D PYTHON_EXECUTABLE=~/.virtualenvs/cv/bin/python \
              -D BUILD_EXAMPLES=ON ..

            make -j4 # From build directory execute make, it is recommended to do this in several threads # runs 4 jobs in parallel

            # Now link OpenCV4 into python3 virtual environment

            $ cd ~/.virtualenvs/cv/lib/python3.5/site-packages/
            $ ln -s /usr/local/python/cv2 cv2
            $ cd
Step 4 : Verification of Installation: Now we can verify OpenCV4 version
            $ workon cv
            $ python
            >>> import cv2
            >>> cv2.__version__   # It will show 4.0.0-dev if downloaded from git, if you have downloaded the stable version then 4.0.0
            >>> quit()
